React is also known as “React.js” or “ReactJS.” React is an open-source JavaScript library that is used for building user interfaces specifically for single-page applications. React renders UI components and supports front-end and back-end. React is the “V” of the Model-View-Controller framework. With the MVC framework, developers can factor out components of an application and easily update them and we can organize the code by grouping their purpose. React is responsible for the view part of the MVC framework. The view part of the framework is all the functions that directly interact with the user.

React can be used to make a simple HTML page interactive and to create complex applications. Nitin Pandit of c-sharpcorner.com stated that “React allows developers to create large web applications that can change data, without reloading the page. The main purpose of React is to be fast, scalable, and simple. It works only on user interfaces in the application.”

Around 2011, the developers at Facebook started having issues with code maintenance. The Facebook Ads application was having a lot of cascading updates. Facebook’s code needed to be updated to be more efficient to handle the cascading updates and at the same time provide reasonably good user experience. Jordan Walke, a former employee of Facebook, created React. But Walke released an early prototype of React called "FaxJS,” which was deployed in 2011 on Facebook’s News Feed and 2012 on Instagram.

On the reactjs.org homepage you’ll notice the word “Declarative.” To clarify, React is declarative because it doesn’t tell a browser to do something. React tells the browser what the developer would like to happen and let’s the browser figure out how to do it. React has a helpful developer toolset which is its open-source library. React is easy to learn. But it’s better if you learn the essentials of programming, HTML, CSS, JavaScript. You’re basically telling the application what you would like to happen and not how you would like to represent a state. React has a huge and strong community of developers. On the reactjs.org website they feature Stack Overflow, and a list of popular discussion forums on DEV, Hashnode, Reactiflux, Reddit, and Spectrum. React is globally popular. Many well-known companies use this framework. Examples are Facebook, Instagram, Netflix, Reddit, BBC, UberEats, and Airbnb.

With React the UI occurs on the browser. In React you must split up the UI components. This works because each component keeps the code necessary to update and display the UI and helps to make the application easier to understand, and components are reusable. With React we have to define a constant to hold the input value. And this constant is set when the input changes. The application doesn't depend on the DOM. The application stores the entire current state in a JavaScript variable. React intuitively updates UI based on a set variable and changing state within a component.
